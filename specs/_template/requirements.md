## Requisitos funcionales

- Lista corta de capacidades verificables (no implementaciones).
- Ej: "el usuario puede iniciar sesión con email + password".

## Requisitos no funcionales

- UX (idioma, accesibilidad, tema).
- Performance (latencia esperada, offline, etc.).
- Seguridad (auth, almacenamiento sensible).

## Fuera de scope

Lista explícita de cosas que **no** entran en este spec. Sirve para parar el alcance creciente.
